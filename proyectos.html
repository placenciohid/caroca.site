Analisis de Datos Clinicos

Usando los datos del dia a dia para entender y mejorar la atencion del Programa de la Mujer.

La pregunta

En Atencion Primaria registramos cientos de atenciones cada mes — controles prenatales, postparto, screening oncologico, talleres educativos. Pero esos datos quedan archivados sin ser analizados.

Que nos pueden decir sobre como estamos atendiendo a nuestras pacientes?

Explore cada area del Programa de la Mujer que analice con datos reales, consultas SQL y visualizaciones:

Prenatal: Contexto clinico

Toda gestante en control debe recibir dos vacunas: Influenza y DTPA. No es opcional — es una meta sanitaria nacional. El problema no es que no se registre el dato; el problema es que nadie lo estaba usando.

Cuando revise los datos, la pregunta no era "cuantas gestantes tenemos". Era: quien esta pendiente y quien se esta quedando sin tiempo. Una gestante sobre 28 semanas sin DTPA ya es una urgencia.

Influenza tiene brechas. Pero DTPA es otra escala — 87% de las gestantes pendientes:

Esta consulta cuenta, para cada vacuna, cuantas gestantes la tienen al dia y cuantas estan pendientes. El UNION ALL combina ambas vacunas en una sola tabla.

-- Cobertura por vacuna: al dia vs. pendiente SELECT 'Influenza' AS vacuna, SUM(CASE WHEN vacuna_influenza THEN 1 ELSE 0 END) AS vacunada, SUM(CASE WHEN NOT vacuna_influenza THEN 1 ELSE 0 END) AS pendiente FROM controles_prenatales UNION ALL SELECT 'DTPA', SUM(CASE WHEN vacuna_DTPA THEN 1 ELSE 0 END), SUM(CASE WHEN NOT vacuna_DTPA THEN 1 ELSE 0 END) FROM controles_prenatales

Ahora filtro solo a las gestantes sobre 28 semanas que aun no se vacunan — las mas urgentes.

-- Gestantes >28 sem sin vacuna: grupo de rescate urgente SELECT rut_paciente, EG_ingreso, vacuna_influenza, vacuna_DTPA FROM controles_prenatales WHERE (NOT vacuna_influenza OR NOT vacuna_DTPA) AND EG_ingreso > 28 ORDER BY EG_ingreso DESC
Hallazgo: De 123 gestantes, 47 no tenian Influenza (38%). Pero la brecha real esta en DTPA: 107 de 123 (87%) pendientes. Al cruzar con edad gestacional, se identificaron pacientes sobre 28 semanas sin ninguna de las dos vacunas — el grupo de rescate mas urgente. Recomendacion: generar listado semanal automatizado para que el equipo de matroneria contacte directamente a estas pacientes antes de que se les pase el plazo.

Postparto: Contexto clinico

La lactancia materna exclusiva (LME) es el estandar de oro en el control de puerperio. Las guias de la OMS y del Ministerio de Salud la recomiendan hasta los 6 meses. El sistema tambien registra la presencia del padre en el control como indicador de apoyo familiar.

Con una muestra inicial pequena — los primeros controles registrados — ya se puede validar que el sistema funciona y observar la distribucion real de tipos de alimentacion.

Aun con una muestra inicial, el sistema ya captura la variedad real de alimentacion — desde LME hasta formula exclusiva:

Agrupa todos los controles por tipo de alimentacion y cuenta cuantos hay de cada uno.

-- Distribucion por tipo de alimentacion SELECT tipo_de_alimentacion, COUNT(*) AS n FROM controles_puerperio WHERE tipo_de_alimentacion IS NOT NULL GROUP BY 1 ORDER BY n DESC

Cruza el tipo de alimentacion con la presencia del padre para ver si hay relacion entre ambos.

-- Alimentacion vs. presencia del padre SELECT tipo_de_alimentacion, SUM(CASE WHEN presencia_del_padre THEN 1 ELSE 0 END) AS con_padre, SUM(CASE WHEN NOT presencia_del_padre THEN 1 ELSE 0 END) AS sin_padre FROM controles_puerperio WHERE tipo_de_alimentacion IS NOT NULL GROUP BY 1
Hallazgo: Con 6 registros iniciales, aun no podemos sacar conclusiones estadisticas — pero el sistema ya esta listo. Se observa un caso por cada tipo de alimentacion, lo que confirma que el registro esta capturando la variedad real. A medida que se acumulen controles, este analisis automaticamente mostrara si la presencia del acompanante se asocia con mayores tasas de LME. Recomendacion: mantener el registro activo y reforzar la invitacion al padre en cada control de puerperio.

Oncologia: Contexto clinico

Esta es el area mas critica. Un PAP alterado o una mamografia alterada deben gatillar una derivacion a nivel secundario. Sin excepciones. Cada dia sin derivacion es un riesgo directo para la seguridad de la paciente.

La pregunta que quise responder fue simple: siempre se derivo a quien correspondia? Y si se derivo, cuanto se demoro el proceso.

99% suena bien. Pero en oncologia, cada caso sin derivar es una paciente en riesgo:

Cuenta, para cada tipo de examen alterado, cuantas pacientes fueron derivadas y cuantas no.

-- Derivacion efectiva vs. sin derivacion por examen SELECT tipo_de_examen_alterado, SUM(CASE WHEN fecha_derivacion_nivel_secundario IS NOT NULL THEN 1 ELSE 0 END) AS derivada, SUM(CASE WHEN fecha_derivacion_nivel_secundario IS NULL THEN 1 ELSE 0 END) AS sin_derivacion FROM controles_oncologicos GROUP BY 1

Para las que si fueron derivadas, calcula cuantos dias en promedio pasaron entre el examen y la derivacion.

-- Tiempo de derivacion en dias (promedio y maximo) SELECT tipo_de_examen_alterado, AVG(fecha_derivacion_nivel_secundario - fecha_de_examen_alterado) AS dias_promedio, MAX(fecha_derivacion_nivel_secundario - fecha_de_examen_alterado) AS dias_maximo FROM controles_oncologicos WHERE fecha_derivacion_nivel_secundario IS NOT NULL GROUP BY 1
Hallazgo critico: De 223 PAP alterados, 221 fueron derivados correctamente. De 191 mamografias alteradas, 189 fueron derivadas. Pero esas 4 pacientes sin derivacion son una alerta directa de seguridad: cada una necesita seguimiento inmediato. Recomendacion: generar listado inmediato para la jefatura clinica con las pacientes sin derivacion, e implementar una alerta automatica que avise cuando un examen alterado lleve mas de 48 horas sin registro de derivacion.

Conclusion

Este proyecto muestra que con herramientas simples — una base de datos, SQL y Python — se pueden transformar registros clinicos en informacion que mejora la atencion.

No se necesita ser informatica. Solo curiosidad por los datos y ganas de mejorar la atencion de las pacientes.
